StorBox如何保证数据安全

<<返回帮助中心

StorBox的安全特性如下：

  • 访问安全
    • StorBox提供登录失败锁定机制，防止暴力方式破解您的登录密码。
    • StorBox提供访问文件和文件夹链接的提取码，防止非法用户访问文件和文件夹。
    • StorBox对所有URL（Uniform Resource Locator）采用了权限划分，特定URL需要用户具有特定权限或角色才能访问。用户认证通过后，根据用户信息进行角色和权限授权，每次URL访问需要校验授权信息，无法越权访问。
    • StorBox提供安全访问控制，即通过用户类型、用户访问时所处的网络和文件类别等控制要素来决定用户能否对文件进行操作的一种控制策略。企业管理员可以根据本企业的安全访问规则统一对本系统的数据安全访问进行控制。
    • StorBox采用对象存储机制，数据均以对象的形式存储于指定的容器中，必须通过授权码和签名密钥访问。各容器之间逻辑隔离，保证了数据访问的安全性。
  • 传输安全

    StorBox支持数据传输加密，支持通过HTTPS（Hypertext Transfer Protocol Secure）通道方式传输数据，从而保证数据在传输环节不被破坏和影响。

  • 存储安全
    • 数据分片存储

      数据被分片后，通过分布式Hash算法随机保存在不同的存储节点、不同的硬盘上，恶意用户无法利用单个存储节点或硬盘获取您的数据。当业务生命周期结束或用户自行终止业务的使用时，数据分片存储技术支持数据的安全销毁，您的数据不会被泄露。

    • 用户数据隔离

      每个用户拥有独立的逻辑空间和存储策略，用户数据完全隔离，实现用户数据安全，确保用户数据的私密性。

    • 用户敏感数据加密存储

      用户敏感数据、隐私数据（如鉴权信息、邮箱信息等）采用AES（Advanced Encryption Standard）算法加密后存储。
